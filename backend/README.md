# Video Scripts API v2

FastAPI backend for the Video Scripts YouTube script generation platform.

## Stack

- **Framework:** FastAPI + Uvicorn / Gunicorn
- **Database:** PostgreSQL (async via SQLAlchemy 2 + asyncpg)
- **Migrations:** Alembic
- **Auth:** JWT (access + refresh), Google OAuth, magic-link email
- **AI:** OpenAI (GPT-4.1) — title, outline, and full script generation
- **Rate limiting:** slowapi
- **Storage:** AWS S3 (optional, falls back to local)
- **Affiliates:** Tolt

## Local Setup

### 1. Clone and enter the directory

```bash
git clone <repo-url>
cd backend_v2
```

### 2. Create and activate a virtual environment

```bash
python3 -m venv .venv
source .venv/bin/activate       # Linux / macOS
# .venv\Scripts\activate        # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Copy the example and fill in your values:

```bash
cp .env.example .env
```

Minimum required variables:

```env
# Core
SECRET_KEY=your-secret-key-min-32-chars
DEBUG=true
ENVIRONMENT=development

# Database (use one of the two options below)
DATABASE_URL=postgresql://user:password@localhost:5432/video_scripts
# — or — individual vars:
# DATABASE_USER=postgres
# DATABASE_PASSWORD=password
# DATABASE_NAME=video_scripts
# DATABASE_HOST=localhost
# DATABASE_PORT=5432

# OpenAI
OPENAI_API_KEY=sk-...
OPENAI_MODEL=gpt-4.1

# Google OAuth (optional — only needed for /auth/google)
GOOGLE_OAUTH2_CLIENT_ID=...
GOOGLE_OAUTH2_CLIENT_SECRET=...

# Brevo email (optional — only needed for magic-link emails)
BREVO_API_KEY=...

# Tolt affiliate (optional)
TOLT_API_KEY=...

# AWS S3 (optional — leave blank to use local storage)
USE_S3_FOR_STORAGE=false
AWS_ACCESS_KEY_ID=...
AWS_SECRET_ACCESS_KEY=...
AWS_STORAGE_BUCKET_NAME=...
AWS_S3_REGION_NAME=us-east-1
```

### 5. Create the database

```bash
createdb video_scripts       # PostgreSQL CLI
# or via psql:
# CREATE DATABASE video_scripts;
```

### 6. Run database migrations

```bash
alembic upgrade head
```

### 7. Start the development server

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

API docs will be available at:
- Swagger UI: http://localhost:8000/api/docs
- ReDoc: http://localhost:8000/api/redoc
- OpenAPI JSON: http://localhost:8000/api/openapi.json

## Alembic Migration Commands

| Command | Description |
|---------|-------------|
| `alembic upgrade head` | Apply all pending migrations |
| `alembic downgrade -1` | Roll back one migration |
| `alembic downgrade base` | Roll back all migrations |
| `alembic revision --autogenerate -m "description"` | Generate a new migration from model changes |
| `alembic history` | Show migration history |
| `alembic current` | Show the currently applied revision |
| `alembic show <revision>` | Show the SQL for a specific revision |

> **Important:** Always review autogenerated migration files before applying them.
> Alembic cannot detect all changes (e.g. column type changes, constraints) reliably.

## API Endpoints

### Auth — `/api/auth`

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/signup` | Register with email + password |
| `POST` | `/google` | Login / register with Google ID token |
| `POST` | `/magic-link` | Send magic-link login email |
| `POST` | `/magic-link/verify` | Verify magic-link token → JWT pair |
| `POST` | `/admin/login` | Admin login (email + password) |
| `POST` | `/refresh` | Exchange refresh token for new token pair |
| `POST` | `/logout` | Invalidate refresh token |

### Users — `/api/v1/users`

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/me` | Get current user profile |
| `PATCH` | `/me` | Update profile (fullname, username, language, avatar) |
| `GET` | `/me/settings/notification` | Get notification preferences |
| `PATCH` | `/me/settings/notification` | Update notification preferences |
| `GET` | `/me/settings/privacy` | Get privacy preferences |
| `PATCH` | `/me/settings/privacy` | Update privacy preferences |

### Scripts — `/api/v1/scripts`

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/tones` | List available tones |
| `GET` | `/template-styles` | List template styles (short / medium / long) |
| `POST` | `/titles/generate` | Generate YouTube titles from a prompt |
| `POST` | `/outlines/generate` | Generate a script outline |
| `GET` | `/outlines` | List user's outlines |
| `GET` | `/outlines/{id}` | Get outline detail |
| `DELETE` | `/outlines/{id}` | Delete outline |
| `PATCH` | `/outlines/{id}/status` | Update outline status |
| `POST` | `/outlines/{id}/script` | Generate full script from outline |
| `GET` | `/scripts` | List user's scripts |
| `GET` | `/scripts/{id}` | Get script detail |
| `DELETE` | `/scripts/{id}` | Delete script |
| `PATCH` | `/scripts/{id}/status` | Update script status |

### Niches — `/api/v1/niches`

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | List active niches |
| `GET` | `/{id}` | Get niche detail |

### Notifications — `/api/v1/notifications`

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | List notifications (filter: `is_read`, `notification_type`) |
| `POST` | `/{id}/read` | Mark notification as read |
| `POST` | `/read-all` | Mark all notifications as read |

### Affiliates — `/api/v1/affiliates`

| Method | Path | Description | Auth |
|--------|------|-------------|------|
| `POST` | `/click` | Track referral link click | None |

### Admin — `/api/v1/admin` *(admin role required)*

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/stats` | Platform-wide statistics |
| `GET` | `/user-stats` | User breakdown statistics |

| `POST` | `/users/{id}/activate` | Activate user |
| `POST` | `/users/{id}/deactivate` | Deactivate user |
| `PATCH` | `/users/{id}/role` | Change user role |
| `GET` | `/niches` | List all niches (including inactive) |
| `POST` | `/niches` | Create niche |
| `GET` | `/niches/{id}` | Get niche detail |
| `PATCH` | `/niches/{id}` | Update niche |
| `DELETE` | `/niches/{id}` | Delete niche |

## Rate Limits

| Endpoint | Limit | Key |
|----------|-------|-----|
| `POST /auth/signup` | 5 / hour | IP |
| `POST /auth/magic-link` | 5 / 10 min | IP |
| `POST /auth/magic-link/verify` | 10 / min | IP |
| `POST /auth/google` | 20 / min | IP |
| `POST /auth/refresh` | 30 / min | IP |
| `POST /scripts/titles/generate` | 10 / min | User |
| `POST /scripts/outlines/generate` | 5 / min | User |
| `POST /scripts/outlines/{id}/script` | 3 / min | User |

Authenticated endpoints are keyed by user ID so different users on the same IP (e.g. office network) don't affect each other. Rate-limited responses return HTTP `429` with a `Retry-After` header.

## Deployment on Railway

### Required environment variables

Set these in your Railway service's **Variables** tab:

```
SECRET_KEY
DATABASE_URL          # Railway provides this automatically when you add a Postgres plugin
OPENAI_API_KEY
GOOGLE_OAUTH2_CLIENT_ID
GOOGLE_OAUTH2_CLIENT_SECRET
BREVO_API_KEY
TOLT_API_KEY
ENVIRONMENT=production
DEBUG=false
```

### How it deploys

The `railway.json` at the project root configures the deployment:

```json
{
  "build": { "builder": "NIXPACKS" },
  "deploy": {
    "startCommand": "alembic upgrade head && gunicorn app.main:app -c gunicorn.conf.py",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
```

On every deploy Railway will:
1. Install dependencies from `requirements.txt` (via Nixpacks)
2. Run `alembic upgrade head` to apply any pending migrations
3. Start the app with Gunicorn + UvicornWorker

### Manual deploy steps

```bash
# Install Railway CLI
npm install -g @railway/cli

# Login and link project
railway login
railway link

# Deploy
railway up

# Run a one-off command (e.g. check migration status)
railway run alembic current
```
